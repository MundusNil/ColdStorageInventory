# 冷库库存主流程使用说明

日期：2026-07-04

## 使用前准备

确认以下条件已经满足：

1. 后端服务已启动。
2. 前端服务已启动。
3. 当前账号已登录。
4. 当前账号有库存查看和库存修改权限。
5. 系统中已有货品、库位和库存批次。

## 入口

打开冷库工作台：

```text
/cold-storage/
```

项目首页当前会默认跳转到冷库工作台。

## 查库存

1. 在“查库存”输入框中输入货品、库位或批次号。
2. 默认只显示有库存的批次。
3. 可以点击“刷新”重新读取库存数据。
4. 页面最多显示前 24 条结果，继续输入关键词可以缩小范围。

库存卡片会显示：

- 货品名称。
- 当前状态。
- 当前数量和单位。
- 库位。
- 批次号。
- 到期日期。

## 新货入库

1. 点击工作台上方的“新货入库”卡片按钮。
2. 在弹窗里选择货品。
3. 填写入库数量。
4. 选择库位。
5. 可选填写批次号。
6. 可选填写到期日期。
7. 可选填写进价、币种、包装/单位说明。
8. 点击“确认入库”。

提交成功后：

- 系统会创建真实库存批次。
- 库存卡片自动刷新。
- 可以通过货品名、库位或批次号搜索到新库存。

注意：

- 如果货品主档配置了默认库位或默认保质期，后端可能会自动补全库位或到期日期。
- 这个入口保留了常用入库字段，隐藏了供应商货品、序列号、标签等复杂字段。
- 如需完整高级字段，仍可使用 InvenTree 原有库存页面。

## 出库扣数

1. 在库存卡片中找到要出库的批次。
2. 点击该卡片上的“出库扣数”。
3. 弹窗会自动带出货品、批次号、出库库位和当前库存。
4. 填写客户/去向。
5. 填写出库数量。
6. 可选填写单价，系统会自动计算总价。
7. 可选填写备注，例如司机、车牌、纸质单号、临时说明。
8. 点击“确认出库扣数”。

提交成功后：

- 系统会真实扣减该库存批次。
- 弹窗自动关闭。
- 库存卡片自动刷新。
- 页面右上角出现成功提示。

## 改库位

1. 在库存卡片中找到要移动的批次。
2. 点击该卡片上的“改库位”。
3. 弹窗会自动带出货品、批次号、原库位和当前库存。
4. 选择新库位。
5. 可选填写移动数量；留空表示整批移动。
6. 可选填写备注，例如临时腾库、串库整理。
7. 点击“确认改库位”。

提交成功后：

- 系统会真实移动该库存批次。
- 弹窗自动关闭。
- 库存卡片自动刷新。

## 盘点改数

1. 在库存卡片中找到要盘点的批次。
2. 点击该卡片上的“盘点改数”。
3. 弹窗会自动带出货品、批次号、库位和账面数量。
4. 填写现场实际数量。
5. 选择差异原因。
6. 可选填写备注。
7. 点击“确认盘点改数”。

提交成功后：

- 系统会把该库存批次同步为实际数量。
- 弹窗自动关闭。
- 库存卡片自动刷新。

## 报损/过期

1. 在库存卡片中找到要报损或过期处理的批次。
2. 点击该卡片上的“报损/过期”。
3. 弹窗会自动带出货品、批次号、库位和当前库存。
4. 填写报损数量。
5. 选择原因。
6. 可选填写备注，例如拍照、主管确认、处理方式。
7. 点击“确认报损/过期”。

提交成功后：

- 系统会真实扣减该库存批次。
- 弹窗自动关闭。
- 库存卡片自动刷新。

## 库存流水备注写入规则

本次不会创建销售单、报损审批单或独立盘点单。客户、价格、原因和说明信息会写入库存流水备注。

出库备注格式大致如下：

```text
冷库出库扣数
货品：xxx
批次号：xxx
出库库位：xxx
出库数量：5
客户/去向：xxx
出库日期：2026-07-04
单价：10
总价：50
备注：xxx
```

后续需要对客户、价格、利润做统计时，再把这些信息从备注升级为结构化销售记录。

## 常见失败提示

“请先从库存卡片选择要出库的批次。”

- 需要从库存卡片点“出库扣数”，不要直接打开空表单。

“请填写客户或去向。”

- 客户/去向为必填项。

“请填写大于 0 的出库数量。”

- 出库数量不能为空、0 或负数。

“出库数量不能大于当前库存。”

- 出库数量超过了当前批次可见库存。

“出库扣数失败，请检查数量、权限和后端服务状态。”

- 可能是当前账号没有库存修改权限。
- 可能是后端服务没有启动。
- 可能是库存状态被后端判定为不可扣减。
- 可能是其他人已经改动该批次，页面库存不是最新值。

“库位列表读取失败，请确认后端服务和权限。”

- 可能是当前账号没有库位查看权限。
- 可能是后端服务没有启动。

“改库位失败，请检查库位、数量、权限和后端服务状态。”

- 可能是目标库位无效。
- 可能是移动数量超过当前库存。
- 可能是当前账号没有库存修改权限。

“盘点改数失败，请检查数量、权限和后端服务状态。”

- 可能是实际数量无效。
- 可能是当前账号没有库存修改权限。

“报损/过期失败，请检查数量、权限和后端服务状态。”

- 可能是报损数量超过当前库存。
- 可能是当前账号没有库存修改权限。

## 测试建议

正式数据录入前，建议先用测试货品验证。

推荐入库测试步骤：

1. 准备一个测试货品和测试库存批次，例如库存数量 10。
2. 点击“新货入库”。
3. 选择测试货品。
4. 填写数量 10。
5. 选择测试库位。
6. 填写测试批次号。
7. 提交。
8. 搜索测试货品，确认能看到新库存。

推荐出库测试步骤：

1. 使用刚入库的测试库存批次。
2. 进入冷库工作台并搜索该货品。
3. 点击“出库扣数”。
4. 填写客户/去向为“测试客户”。
5. 出库数量填写 2。
6. 单价填写 5。
7. 提交。
8. 确认库存卡片数量从 10 变成 8。
9. 进入原 InvenTree 库存详情或库存流水，确认有扣减记录。
10. 再测试一次超量出库，例如输入 999，应被前端拦截。

推荐改库位测试步骤：

1. 使用测试库存批次。
2. 点击“改库位”。
3. 选择另一个测试库位。
4. 移动数量留空。
5. 提交。
6. 确认库存卡片库位变化。

推荐盘点测试步骤：

1. 使用测试库存批次。
2. 点击“盘点改数”。
3. 实际数量填写 7。
4. 差异原因选择“少货”或“录错”。
5. 提交。
6. 确认库存卡片数量同步为 7。

推荐报损测试步骤：

1. 使用测试库存批次。
2. 点击“报损/过期”。
3. 报损数量填写 1。
4. 原因选择“破损”或“过期”。
5. 提交。
6. 确认库存卡片数量减少 1。

## 提交前验证

前端改动提交前，请在本机手动执行：

```bash
cd src/frontend
yarn harness
yarn tsc --noEmit
```

如果涉及语言包，再手动执行：

```bash
yarn compile
```

Codex 不会代跑这些前端命令。
