# 冷库库存主流程需求说明

日期：2026-07-04

## 背景

冷库工作台当前采用渐进改造策略：保留 InvenTree 原有库存模型和权限体系，在旁边新增更适合现场人员使用的中文冷库入口。此前冷库页面已经具备库存卡片查询和多种业务表单草稿，本轮把冷库日常库存主流程接入现有真实库存接口。

本次需求打通以下主流程：

1. “新货入库 -> 创建真实库存批次 -> 刷新库存卡片”。
2. “从库存卡片选择批次 -> 填写出库信息 -> 真实扣减库存 -> 刷新库存卡片”。
3. “从库存卡片选择批次 -> 选择新库位 -> 真实移动库存 -> 刷新库存卡片”。
4. “从库存卡片选择批次 -> 填写实际数量 -> 真实盘点改数 -> 刷新库存卡片”。
5. “从库存卡片选择批次 -> 填写报损原因和数量 -> 真实扣减库存 -> 刷新库存卡片”。

## 目标

1. 操作人员可以在冷库工作台查询库存批次。
2. 操作人员可以通过“新货入库”创建真实库存批次。
3. 操作人员可以从某一张库存卡片进入出库扣数表单。
4. 出库提交后系统真实扣减该库存批次数量。
5. 操作人员可以从库存卡片进入改库位、盘点改数、报损/过期表单。
6. 客户、单价、总价、日期、原因和备注写入库存流水备注，便于后续追溯。
7. 任一库存动作成功后页面刷新库存卡片，避免继续显示旧库存。

## 当前范围

已接入真实库存：

- 查库存：读取现有库存 API。
- 新货入库：提交到现有库存创建 API。
- 出库扣数：提交到现有库存扣减 API。
- 改库位：提交到现有库存移动 API。
- 盘点改数：提交到现有库存盘点 API。
- 报损/过期：复用现有库存扣减 API，并写入报损原因备注。

## 新货入库业务规则

新货入库使用 InvenTree 现有库存批次创建能力，冷库工作台只负责提供中文入口和更贴近现场的字段文案。

常用字段：

- 货品。
- 数量。
- 库位。
- 批次号。
- 到期日期。
- 进价。
- 进价币种。
- 包装/单位说明。

隐藏字段：

- 供应商货品。
- 包装换算。
- 序列号。
- 自定义状态。
- 标签。
- 外部链接。
- 所有人。
- 耗尽后删除。

这些字段暂不暴露给现场入口，避免表单过复杂；原 InvenTree 通用页面仍可使用完整能力。

## 出库扣数业务规则

出库扣数必须从库存卡片进入，不能手动输入库存批次 ID。

必填字段：

- 客户/去向。
- 出库数量。

自动带出字段：

- 货品。
- 批次号。
- 出库库位。
- 当前库存。
- 出库日期，默认当天。

可选字段：

- 单价。
- 备注。

自动计算字段：

- 总价 = 出库数量 x 单价。

## 改库位业务规则

改库位必须从库存卡片进入，不能手动输入库存批次 ID。

必填字段：

- 新库位。

自动带出字段：

- 货品。
- 批次号。
- 原库位。
- 当前库存。
- 移动日期，默认当天。

可选字段：

- 移动数量。不填表示整批移动。
- 备注。

## 盘点改数业务规则

盘点改数必须从库存卡片进入，不能手动输入库存批次 ID。

必填字段：

- 实际数量。
- 差异原因。

自动带出字段：

- 货品。
- 批次号。
- 库位。
- 账面数量。
- 差异数量。
- 盘点日期，默认当天。

可选字段：

- 备注。

## 报损/过期业务规则

报损/过期必须从库存卡片进入，不能手动输入库存批次 ID。

必填字段：

- 报损数量。
- 原因。

自动带出字段：

- 货品。
- 批次号。
- 库位。
- 当前库存。
- 处理日期，默认当天。

可选字段：

- 备注。

## 校验规则

1. 未选择库存批次时不能提交。
2. 客户/去向不能为空。
3. 出库数量必须大于 0。
4. 如果前端能读取当前库存，出库数量不能大于当前库存。
5. 改库位必须选择新库位，移动数量留空时按当前整批库存移动。
6. 移动数量不能大于当前库存。
7. 盘点实际数量不能小于 0。
8. 盘点必须选择差异原因。
9. 报损数量必须大于 0，且不能大于当前库存。
10. 报损必须选择原因。
11. 提交中禁用关闭按钮，避免重复操作造成状态混乱。

## 接口映射

前端提交地址：

```text
POST /api/stock/
POST /api/stock/remove/
POST /api/stock/transfer/
POST /api/stock/count/
```

前端枚举：

```text
ApiEndpoints.stock_item_list
ApiEndpoints.stock_remove
ApiEndpoints.stock_transfer
ApiEndpoints.stock_count
```

入库提交数据结构由现有 `useStockFields` 和 `useCreateApiFormModal` 生成，最终提交到 `stock_item_list`。

出库提交数据结构：

```json
{
  "items": [
    {
      "pk": 123,
      "quantity": 5
    }
  ],
  "notes": "冷库出库扣数\n货品：...\n批次号：...\n客户/去向：..."
}
```

移库提交数据结构：

```json
{
  "items": [
    {
      "pk": 123,
      "quantity": 5
    }
  ],
  "location": 456,
  "notes": "冷库改库位\n货品：...\n新库位：..."
}
```

盘点提交数据结构：

```json
{
  "items": [
    {
      "pk": 123,
      "quantity": 8
    }
  ],
  "notes": "冷库盘点改数\n货品：...\n实际数量：..."
}
```

报损/过期复用出库扣减结构，备注中记录报损原因。

后端实际处理：

- 入库使用 InvenTree 现有 `StockList.create()`。
- 出库和报损使用 InvenTree 现有 `StockRemoveSerializer`，调用库存批次的 `take_stock()`。
- 改库位使用 InvenTree 现有 `StockTransferSerializer`，调用库存批次移动逻辑。
- 盘点使用 InvenTree 现有 `StockCountSerializer`，调用库存批次盘点逻辑。
- 所有操作均生成库存流水。

## 权限要求

用户需要具备库存查看权限，才能读取库存卡片。

用户需要具备库存修改权限，才能成功提交新货入库和出库扣数。权限不足时后端会拒绝请求，前端显示失败提示。

## 非目标

本次不做以下内容：

- 不新增销售单。
- 不新增客户应收账款。
- 不新增冷库专用后端模型。
- 不改变 InvenTree 原有库存创建、扣减、移动和盘点逻辑。
- 不删除现有 InvenTree 通用页面和能力。

## 验收标准

1. 进入冷库工作台后，可以看到库存卡片。
2. 点击“新货入库”，可以打开真实库存批次创建弹窗。
3. 新货入库成功后，库存卡片刷新并能查到新增库存。
4. 点击某张库存卡片上的“出库扣数”，弹出出库扣数单。
5. 出库表单能正确带出货品、批次号、库位和当前库存。
6. 未填客户/去向时无法提交。
7. 出库数量为 0、空值或负数时无法提交。
8. 出库数量超过当前库存时无法提交。
9. 合法提交后，库存批次数量减少。
10. 出库提交成功后弹窗关闭，库存卡片刷新。
11. 库存流水备注包含客户、数量、日期、价格和备注信息。
12. 点击某张库存卡片上的“改库位”，可以选择新库位并真实移动库存。
13. 点击某张库存卡片上的“盘点改数”，可以录入实际数量并同步库存。
14. 点击某张库存卡片上的“报损/过期”，可以按原因真实扣减库存。
15. 任一动作成功后库存卡片刷新。

## 风险和后续

当前客户、价格和总价只写入库存流水备注，不进入结构化销售/财务模型。后续如果要做销售日报、客户欠款或利润统计，需要独立设计销售出库单或客户流水模型。

建议下一阶段按以下顺序继续：

1. 销售/客户/价格结构化记录。
2. 报损审批和照片附件。
3. 盘点复核流程。
4. 冷库固定库位模板和库位快捷选择。
