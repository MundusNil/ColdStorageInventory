# 页面与业务流程设计

整理时间：2026-06-29 12:01

本文档只写冷库工作台的页面和业务流程。总体方向见 `20260629-120114_01_冷库系统总方案.md`，历史记录见 `20260629-120114_03_开发记录与日期索引.md`。

## 页面总览

第一版页面入口：

```text
冷库工作台

常用操作：
[新货入库] [出库扣数] [查库存]
[改库位]   [盘点改数] [报损/过期]

提醒区域：
- 快过期提醒
- 库存不足提醒，后续可选
- 今日入库 / 今日出库，后续可选
```

页面语言要全部用冷库业务词，不暴露 IPN、BOM、生产订单、内部 API 等术语。

## 货品主档

### 作用

货品主档是总清单表，用来减少重复输入。

入库时老板输入货品名称，系统自动从主档带出固定信息。

### 字段

必填：

- 货品名称
- 单位
- 保质期

建议字段：

- 品牌名
- 默认供应商
- 默认库位
- 备注

### 页面行为

- 支持按货品名称搜索。
- 如果入库时找不到货品，第一版建议提示先建货品主档。
- 后续可以支持在入库弹窗里快速新增货品。

## 新货入库

### 场景

新到一批货，把这批货的数量、价格、生产日期、供货商和库位录进去。

### 字段

必填：

- 货品
- 数量
- 进价
- 生产日期
- 供货商

系统自动：

- 入库日期
- 操作人
- 批次
- 到期日期
- 单位，来自货品主档

可选：

- 存放位置
- 备注

### 批次规则

第一版建议自动生成批次，不要求手动填。

格式可以先简单一些：

```text
货品名-生产日期-序号
```

例如：

```text
大虾-20260601-001
```

如果供货商对区分批次很重要，后续再把供货商拼进去。

### 保存结果

保存后应产生一条库存记录：

- 货品
- 批次
- 当前数量
- 进价
- 生产日期
- 到期日期
- 供货商
- 库位

第一版可以先只实现表单和校验，真实写入要等字段确认后再接 API。

## 出库扣数

### 场景

货卖出或拿走后，在系统里扣掉数量，并记录客户和价格。

### 字段

必填：

- 货品
- 客户
- 单价
- 数量
- 总价

系统自动：

- 出库日期
- 操作人
- 总价，按单价乘数量计算

建议字段：

- 批次，系统推荐快过期或先进来的批次
- 出库库位
- 备注

### 批次选择

如果同一货品有多个批次，页面不要让老板在一堆复杂编号里选。

推荐显示成：

```text
大虾 | 生产 2026-06-01 | 到期 2027-06-01 | 北-架子下-东 | 剩 20 箱
```

默认可以推荐最早到期的批次。

### 扣数规则

第一版建议：

- 不允许出库数量超过当前库存。
- 总价自动计算，但允许老板手动改，后续再决定。
- 出库后减少对应批次库存。

## 查库存

### 当前状态

查库存已经接入为卡片查询，继续复用 InvenTree `stock_item_list` API。

### 搜索方式

第一版保留一个大搜索框：

- 货品名称
- 库位
- 批次

如果真实数据验证发现一个搜索框不够，再拆成多个筛选项。

### 卡片显示

每张库存卡片优先显示：

- 货品名称
- 当前数量和单位
- 库位
- 批次
- 生产日期，后续补充
- 到期日期
- 状态

### 提醒

后续建议增加：

- 快过期：黄色提示。
- 已过期：红色提示。
- 无库存：默认隐藏，需要时通过开关查看。

## 改库位

### 场景

货搬地方了，把系统里的库位改掉。

### 字段

必填：

- 货品
- 批次
- 原库位
- 新库位
- 移动数量

系统自动：

- 移动日期
- 操作人

可选：

- 备注

### 规则

- 新库位从固定库位里选择。
- 移动数量不能超过当前库存。
- 如果整批移动，可以一键填入当前批次数量。

## 盘点改数

### 场景

每月盘点时，实际数量和系统数量不一致，按现场数量修正。

### 字段

必填：

- 货品
- 批次
- 库位
- 实际数量
- 差异原因

系统显示：

- 系统数量
- 差异数量
- 盘点日期
- 盘点人

差异原因建议固定选项：

- 少货
- 多货
- 录错
- 损耗
- 未知

### 规则

当前确认暂时不需要审批，因为只给老板使用。

后续如果普通工人也能盘点，可以增加班组长确认。

## 报损 / 过期

### 场景

货坏了、过期了、不能卖了，记录原因并扣库存。

### 字段

必填：

- 货品
- 批次
- 数量
- 库位
- 原因

系统自动：

- 处理日期
- 操作人

可选：

- 备注

原因建议固定选项：

- 过期
- 破损
- 化冻
- 污染
- 其他

## 打印单据

用户确认可以加打印。

建议等入库和出库字段稳定后再做，避免单据模板反复改。

第一版打印单据：

- 入库单
- 出库单
- 盘点单

单据上建议包含：

- 单据号
- 日期
- 货品
- 批次
- 数量
- 单价 / 总价，出库单需要
- 供货商 / 客户
- 经手人
- 备注

## 页面优先级

| 优先级 | 页面 / 功能 | 原因 |
| --- | --- | --- |
| 1 | 查库存 | 已有基础，风险低 |
| 2 | 货品主档 | 入库自动带信息依赖它 |
| 3 | 新货入库 | 核心业务 |
| 4 | 出库扣数 | 核心业务，但风险更高 |
| 5 | 快过期提醒 | 批次和到期日稳定后再做 |
| 6 | 打印单据 | 字段稳定后再做 |
| 7 | 盘点、报损、改库位真实写入 | 要更多规则确认 |

## 待确认问题

1. 8 个库位的最终名字是否按“北/南/中 + 东/西 + 架上/架下”整理。
2. 供货商和客户是否需要先做成固定档案，还是第一版允许手动输入。
3. 出库是否默认按最早到期批次扣，还是老板手动选批次。
4. 出库总价是否允许手动改。
5. 单据打印是否需要公司名、电话、签字栏。
