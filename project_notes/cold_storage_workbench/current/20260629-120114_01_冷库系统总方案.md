# 冷库系统总方案

整理时间：2026-06-29 12:01

本文档是冷库工作台改造的主线方案。原始讨论和切片记录仍保留在 `../archive/` 的带日期文件里，日期索引见 `20260629-120114_03_开发记录与日期索引.md`。

## 一句话结论

不把 InvenTree 整个系统硬改成冷库软件，而是在现有系统旁边新增一个并行的“冷库工作台”。

老板和现场人员优先使用冷库工作台完成入库、出库、查库存、改库位、盘点、报损等日常动作；原 InvenTree 页面继续保留，给管理员查错、配置和处理复杂数据。

## 为什么这样做

| 做法 | 优点 | 风险 |
| --- | --- | --- |
| 直接重写整套系统 | 页面可以完全贴合冷库 | 工作量大，容易改坏原系统 |
| 全局汉化和改名 | 看起来更统一 | 影响范围太大，很多 ERP 概念不好一刀切 |
| 并行新增冷库工作台 | 改动可控，旧功能保留，能逐步验证 | 第一阶段会有新旧入口并存 |

推荐第三种：先新增工作台，验证流程顺了，再考虑按角色隐藏复杂导航。

## 使用对象

当前确认：主要给老板使用，后面可能扩展给普通工人。

因此第一版可以显示金额、进价、售价、客户、供应商等信息，不急着做“普通工人隐藏金额”。但权限设计要留余地，避免后续扩展时返工。

## 业务范围

冷库主要管理冻品食材，例如：

- 大虾
- 冻鸡腿、鸭腿
- 火锅丸子
- 炸货
- 其他冻品食材

日常目标不是完整 ERP 流程，而是回答这些问题：

- 来了什么货，多少，多少钱，从谁那里来？
- 放在哪个位置？
- 还有多少库存？
- 哪些快过期？
- 卖给了谁，卖了多少，多少钱？
- 盘点时系统数和实际数差多少？
- 哪些货报损或过期了？

## 核心概念

### 货品主档

需要一个总清单表，用来记录货品固定信息：

- 货品名称
- 品牌名
- 单位
- 保质期
- 默认供应商，后续可选
- 默认库位，后续可选
- 备注

入库时输入货品名后，系统应从货品主档自动带出品牌、单位、保质期等固定信息。

### 批次

需要批次，但不让老板手动维护复杂批次号。

推荐做法：

- 入库时填写货品、数量、进价、生产日期、供货商等信息。
- 系统根据这些信息自动生成批次。
- 查询时仍按货品名称为主，点开后能看到不同批次、生产日期、到期日期和剩余数量。

这样可以支持快过期提醒、先进先出出库和利润核算。

### 保质期和到期日期

货品主档里记录保质期，入库时填写生产日期，系统自动计算到期日期。

例如：

```text
生产日期 + 保质期 = 到期日期
```

后续首页可以加“快过期提醒”。

### 库位

库位建议做成固定选项，不让用户随便输入。

当前理解是冷库按东西南北和上下层划分，后续可整理为 8 个标准位置。名称需要再和现场说法对齐，例如：

- 北-架子上-东
- 北-架子上-西
- 北-架子下-东
- 北-架子下-西
- 南-东
- 南-西
- 中-东
- 中-西

### 金额

因为当前只给老板用，第一版可以保留金额字段：

- 入库进价
- 出库单价
- 出库总价
- 后续利润统计

其中出库总价建议自动计算，减少手算错误。

## 第一阶段目标

第一阶段只做并行工作台，不替换旧页面。

工作台首页保留 6 个入口：

```text
冷库工作台

[新货入库] [出库扣数] [查库存]
[改库位]   [盘点改数] [报损/过期]
```

优先顺序：

1. 查库存，只读，风险最低。
2. 货品主档，用来支撑自动带出信息。
3. 新货入库，生成批次和库存。
4. 出库扣数，按批次扣减库存。
5. 盘点改数，每月使用。
6. 报损/过期，扣库存并留原因。
7. 打印单据，等字段确认后再做。

## 技术策略

第一版优先复用 InvenTree 原有能力：

- `Part`：承载货品主档。
- `StockItem`：承载库存批次、数量、库位、到期日期。
- `StockLocation`：承载冷库库位。
- 库存 API：入库、出库、转库位、盘点、扣库存。

暂时不做：

- 不直接删除原模块。
- 不大改数据库。
- 不先接入扫码。
- 不先做复杂审批流。
- 不强制走完整采购单、销售单、生产单流程。

## 权限方向

当前老板自己用，所以第一版权限可以简单。

但后续建议至少区分：

| 角色 | 页面 |
| --- | --- |
| 老板 / 管理员 | 冷库工作台 + 原 InvenTree 完整页面 |
| 普通工人 | 只看冷库工作台和必要查询页面 |
| 班组长，后续可选 | 能看报表、改错、处理盘点差异 |

权限不能只靠隐藏前端按钮，涉及真实库存写入时后端也要限制。

## 下一步建议

下一步不建议继续写散文档，建议直接围绕页面原型推进：

1. 把冷库工作台页面统一成中文老板视角。
2. 增加货品主档原型。
3. 调整新货入库表单字段：货品、数量、进价、生产日期、供货商、库位。
4. 调整出库扣数字段：货品、客户、单价、数量、总价、批次。
5. 确认 8 个库位的最终命名。
