# 库存搜索体验记录

本轮目标：让冷库工人不必找表格右上角的小搜索框，而是在查库存区域直接看到一个明显的手动搜索入口。

## 改动范围

- 在 `ColdStorageStockTable` 上方增加大搜索框。
- 搜索框占据主要宽度，提示语为“输入货品、库位或批次”。
- 搜索仍然调用原表格状态的 `setSearchTerm`，最终走 API 的 `search` 参数。
- 隐藏原 InvenTreeTable 内置搜索框，避免页面出现两个搜索入口。
- 增加“只看有库存”开关，默认打开。
- 开关直接控制 `stock_item_list` 查询参数里的 `in_stock`。

## 设计理由

- 现场没有条码，搜索是最重要的手动入口。
- 搜索入口必须明显，不能藏在复杂表格工具栏里。
- “只看有库存”默认打开，减少空库存项干扰。
- 仍然保持只读，不新增任何库存写入动作。

## 后续问题

- API 的 `search` 具体搜索哪些字段需要后续用真实数据验证。
- 如果默认搜索不能覆盖库位或批次，后面要改成独立字段筛选。
- 移动端需要确认搜索框和开关是否换行自然。
