# 登录密码泄露风险修复记录

时间：2026-06-28 23:42

## 背景

浏览器密码管理器提示本地站点存在泄露密码风险。排查后确认，普通登录表单会通过 POST body 提交用户名和密码，这本身不是“明文放在 URL”。

真正高风险点是登录页曾支持这种地址：

```text
/login?login=用户名&password=密码
```

这会让密码进入浏览器历史记录、地址栏、代理日志、错误日志或截图中。

## 已修复内容

### 前端

文件：`src/frontend/src/pages/Auth/Login.tsx`

- 删除从 URL 参数 `login` + `password` 自动登录的逻辑。
- 页面加载时如果发现地址栏里带有 `login` 或 `password`，会立刻用 `history.replaceState` 清掉这些参数。
- 正常用户名密码登录表单不受影响。

### 后端

文件：`src/backend/InvenTree/InvenTree/helpers.py`

- 新增 `redact_url(url)`。
- 对敏感 query 参数进行脱敏：
  - `api_key`
  - `auth`
  - `bearer`
  - `key`
  - `login`
  - `pass`
  - `password`
  - `secret`
  - `sesame`
  - `token`

文件：`src/backend/InvenTree/InvenTree/api.py`

- API 404 返回中的 `url` 改为脱敏 URL。

文件：`src/backend/InvenTree/InvenTree/middleware.py`

- 异常记录里的完整 URL 改为脱敏 URL。

## 当前安全边界

- 不再支持通过 URL 自动传密码登录。
- 即使有人手动把密码拼到 URL，前端会尝试清理当前地址栏。
- 后端记录 URL 时会把常见敏感参数替换为 `[redacted]`。

## 仍需注意

- 如果浏览器已经保存过泄露密码提示，可能需要在浏览器密码管理器里删除旧记录或改密码。
- 开发环境如果曾访问过带密码的 URL，浏览器历史记录里可能已有旧痕迹。
- 正式给现场人员使用时，应避免共享管理员账号，给每个人单独建低权限账号。
