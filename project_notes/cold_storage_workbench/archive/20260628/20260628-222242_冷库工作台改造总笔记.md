# 冷库工作台改造笔记

## 先说结论

我不建议直接把 InvenTree 整个项目硬改成冷库系统。

更稳妥的做法是：**在现有项目里并行新增一个“冷库工作台”入口**，先让普通工人只用这个入口完成日常操作；原来的零件、库存、采购、销售、生产等页面先保留，主要给管理员维护数据和查错使用。

这样做的好处是：

- 不推倒重来，现有功能还能用。
- 一线人员不用理解复杂 ERP 术语。
- 后面如果发现流程不对，只改工作台，不影响底层数据。
- 等工作台跑顺了，再考虑隐藏或弱化原来的复杂导航。

## 面向谁改

目标用户不是仓储软件专家，而是冷库现场工作人员。

界面原则：

- 字要少，按钮要直观。
- 不出现“IPN、BOM、生产订单、API、文档”等技术词。
- 不依赖扫码，因为现场没有条码。
- 每个页面只解决一个动作，不让用户在大表格里迷路。
- 错误提示要说人话，比如“数量不能小于 0”，不要只显示系统字段名。

## 现有页面不适配的地方

| 现在的说法 | 冷库里建议叫法 | 处理方式 |
|---|---|---|
| 零件 | 货品 / 品名 | 普通人员界面里改叫货品 |
| 库存 | 库存 | 可以保留 |
| 采购 | 入库 / 进货 | 工作台里叫入库 |
| 销售 | 出库 / 发货 | 工作台里叫出库 |
| 生产 | 暂时隐藏 | 冷库日常一般不用 |
| 生产订单 | 暂时隐藏 | 除非有加工或分装流程 |
| 内部零件编码 IPN | 货品编号 / 内部编号 | 普通人员可隐藏或弱化 |
| 位置 | 冷库 / 库位 / 货架 | 改成现场能理解的叫法 |
| 库存调整 | 改数量 / 盘点改数 | 工作台里用业务叫法 |
| 报废 | 报损 / 过期处理 | 冷库更容易理解 |

## 第一阶段：只加工作台，不改旧模块

新增一个首页模块，名字可以叫：

- 冷库工作台
- 今日操作
- 仓库操作台

推荐先叫 **冷库工作台**。

里面只放 6 个常用动作：

```text
冷库工作台

[新货入库] [出库扣数] [查库存]
[改库位]   [盘点改数] [报损/过期]
```

### 1. 新货入库

给工人看的描述：

> 新到一批货，把品名、数量、库位录进去。

需要的字段初步设想：

- 货品名称
- 数量
- 单位
- 冷库 / 库位
- 到货日期
- 备注

后面可选字段：

- 供应商
- 批次，系统根据入库信息自动生成，页面上可以显示但不要求手动填写
- 保质期 / 到期日期，优先从货品主档自动带出和计算
- 经手人

### 2. 出库扣数

给工人看的描述：

> 货拿走了，在系统里把数量减掉。

需要的字段初步设想：

- 货品名称
- 出库数量
- 从哪个库位出
- 出库原因
- 备注

出库原因可以是：

- 发货
- 领用
- 损耗
- 其他

### 3. 查库存

给工人看的描述：

> 看某个货还剩多少，放在哪个库位。

搜索方式：

- 按货品名称搜索
- 按库位搜索
- 按批次搜索，如果你们需要批次

显示内容要简单：

- 品名
- 当前数量
- 单位
- 库位
- 到期日期，如果有

### 4. 改库位

给工人看的描述：

> 货搬地方了，把系统里的库位改一下。

需要的字段：

- 货品
- 数量
- 原库位
- 新库位
- 备注

### 5. 盘点改数

给工人看的描述：

> 实际数和系统数不一样，按现场数量改。

需要的字段：

- 货品
- 库位
- 系统数量
- 实际数量
- 差异原因

差异原因可以是：

- 少货
- 多货
- 录错
- 损耗
- 未知

### 6. 报损 / 过期

给工人看的描述：

> 货坏了、过期了、不能用了，记录下来并扣库存。

需要的字段：

- 货品
- 数量
- 库位
- 原因
- 备注

原因可以是：

- 过期
- 破损
- 化冻
- 污染
- 其他

## 第二阶段：角色区分

建议至少分两类人：

| 角色 | 看到什么 |
|---|---|
| 普通工人 | 只看到冷库工作台和必要的查询页面 |
| 管理员 | 可以看到原 InvenTree 的完整页面 |

后面如果需要，可以再加一类：

| 角色 | 看到什么 |
|---|---|
| 班组长 | 能看报表、改错、审核盘点差异 |

## 第三阶段：弱化原导航

等冷库工作台确认好用以后，再考虑处理顶部导航。

不是马上删除，而是按角色隐藏：

- 普通工人隐藏：生产、采购、销售、零件等复杂模块。
- 管理员保留：完整导航。
- 库存模块可以保留，但入口文案换成更直观的“查库存”。

## 技术上大概怎么改

先不动数据库结构，优先复用 InvenTree 原有库存能力。

初步改动范围：

1. 新增一个前端页面或仪表盘组件：`冷库工作台`
2. 给 6 个操作做独立入口
3. 每个入口先跳转到现有页面或打开简化表单
4. 后续再根据实际流程，把复杂表格页面包装成简单表单
5. 管理员仍然可以进入原页面处理复杂数据

不建议第一步就做的事：

- 不直接删除生产、采购、销售模块。
- 不全局改所有字段名。
- 不大改数据库。
- 不做扫码逻辑。
- 不先做复杂报表。

## 需要你补充的内容

你可以直接按下面这些问题补充。

### 1. 你们冷库实际管什么货？

例如：

- 食材
- 冻品
- 药品
- 原料
- 半成品
- 成品

基本上都是冻品食材，例如大虾，冻的鸡腿鸭腿，火锅丸子，炸货等等

### 2. 货品有没有批次？

比如同一个品名，不同到货日期要不要分开管。

需要带上批次，但不要让老板手动维护复杂批次号。

推荐做法是：入库时老板正常填写货品、数量、进价、生产日期、供货商等信息，系统根据这些信息自动生成批次。老板平时还是按货品名称查库存，点开后可以看到不同批次、生产日期、到期日期和剩余数量。

这样可以解决两个问题：

- 同一个货品不同生产日期不会混在一起。
- 后面做快过期提醒、先进先出出库、利润核算时更准确。

### 3. 货品有没有保质期或到期日？

如果有，首页可能要加“快过期提醒”。

有，推荐加上

### 4. 库位怎么叫？

例如：

- 1 号冷库
- A 区
- A-01 货架
- 某个房间

以东西南北来称呼，北边有上下两层，上面叫架子上，下面叫架子下，南边没有架子，称呼的时候说东边或者西边即可；中间还可以再分，所以一共是八个位置

### 5. 入库时必须填哪些内容？

请按“必须填”和“可填可不填”分开。

应该有一个总的清单表，用来记录各个货品的基本信息（例如名称，品牌名，保质期）
然后我自己填入货的时候只要一打名字它可以自动的从总的里面检索到我的这个商品并且自动帮我填好一些固定的商品信息。
我自己再填写一些内容，必填的有：数量，进价，生产日期，供货商
可选的有：存放位置
日期什么的自动记录

### 6. 出库时必须填哪些内容？

比如是否必须填客户、用途、领用人。

除了必填的商品信息，必填的还有：客户，单价，数量，总价

### 7. 盘点多久做一次？

每天、每周、每月，还是不固定。

每月

### 8. 有没有审批？

比如普通工人改库存数量以后，是否需要管理员确认。

暂时不需要审批，这个只给老板用

### 9. 是否需要打印单据？

比如入库单、出库单、盘点单。

可以加上

### 10. 普通工人能不能看到金额？

如果不能，工作台里要隐藏价格、采购价、销售价相关字段。

只有老板用

## 当前推荐的下一步

先不要动现有页面。

下一步应该先确认这 6 个入口是否符合现场：

- 新货入库
- 出库扣数
- 查库存
- 改库位
- 盘点改数
- 报损 / 过期

确认后，再做一个并行的“冷库工作台”页面原型。
