# Task Plan: 冷库工作台改造笔记

## Goal

先形成一组中文改造笔记，说明 InvenTree 如何渐进改造成适合冷库一线人员手动录入使用的工作台。

## Phases

- [x] Phase 1: 确认用户诉求和约束。
- [x] Phase 2: 梳理当前页面不适配点。
- [x] Phase 3: 写出冷库工作台改造笔记。
- [x] Phase 4: 补充现有功能摸底和冷库动作映射表。
- [ ] Phase 5: 等用户补充业务流程后，再决定是否开始动代码。

## Key Questions

1. 冷库每天最常用的 3-6 个动作是什么？
2. 是否需要区分普通工人、班组长、管理员三类界面？
3. 入库、出库、盘点是否都只靠手动输入，没有条码和扫码枪？
4. 是否需要保留生产、采购、销售等原 InvenTree 模块给管理员使用？
5. 货品是否需要批次和保质期管理？
6. 出库和报损是否需要审核？

## Decisions Made

- 采用并行开发策略：先新增“冷库工作台”，不直接替换旧页面。
- 旧功能保留：原 InvenTree 模块继续给管理员使用，避免一开始改坏现有流程。
- 一线界面避免 ERP 术语：不用零件、生产订单、IPN、BOM 这类词，改成货品、入库、出库、库位、盘点等业务词。
- 第一版走库存直操路线：优先复用现有库存接口，不先接入完整采购单/销售单/生产单。
