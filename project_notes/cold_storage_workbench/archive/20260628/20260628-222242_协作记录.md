# Notes: 冷库工作台改造

## 已确认

- 使用对象是“对电脑一窍不通的冷库工作人员”。
- 当前系统顶部导航和表格标题偏制造业/ERP，不适合一线人员直接使用。
- 现场没有条码，不能按扫码流程设计，必须支持手动输入。
- 用户偏好稳妥渐进：新功能和旧功能并存，验证后再考虑替换。

## 当前不适配点

- “零件”更适合改成“货品 / 品名”。
- “生产”通常不适合冷库日常操作，除非存在加工、分装、包装流程。
- “采购 / 销售”对工人来说不如“入库 / 出库”直观。
- “IPN / BOM / 生产订单”等字段需要对普通人员隐藏或换成业务叫法。

## 初步方向

- 首页新增“冷库工作台”，把一线人员常用动作做成大按钮或简化入口。
- 原系统模块不删除，管理员仍可通过原导航进入。
- 手动输入优先：入库、出库、查库存、改库位、盘点、报损/过期。
- 后续可以按角色隐藏复杂导航，让普通人员只看到工作台。

## 2026-06-28 功能摸底结论

- 现有库存底层已经覆盖“入库、出库、查库存、转库位、盘点改数、报损扣数”的主要动作。
- 第一版不建议先接入完整采购单、销售单、生产单流程，这些对冷库一线工人太重。
- 推荐新增并行的“冷库工作台”，复用现有 StockItem、StockLocation、Part 和库存操作 API。
- 普通工人页面只保留大入口和简化中文字段，管理员继续保留原 InvenTree 页面。
- 详细映射见 `cold_storage_feature_mapping.md`。
